# Präsentation - Stichpunkte

## Folie 1: Titel

Schulverwaltung: Angular-App und Low-Code/KI-Prototyp

## Folie 2: Arbeitsauftrag

- Angular-App erstellen
- Low-Code/KI-App erstellen
- Projekthandbuch erstellen
- Unterschiede präsentieren

## Folie 3: Projektstruktur

- Backend: `SchoolWeb`
- Angular-App: `SchoolWebFrontend`
- Low-Code-App: `LowCodeApp`
- Dokumentation: `docs`

## Folie 4: Angular-App

- Dashboard
- Schüler anzeigen und speichern
- Lehrer anzeigen
- Räume anzeigen
- Statistik anzeigen
- Daten kommen über HTTP vom Backend

## Folie 5: Code der Angular-App

- `app.ts`: Logik der Oberfläche
- `app.html`: Anzeige im Browser
- `school-api.service.ts`: Verbindung zum Backend
- `school.models.ts`: Datentypen

## Folie 6: Low-Code/KI-App

- einfacher Prototyp
- HTML, CSS und JavaScript
- Schüler hinzufügen/löschen
- Speicherung mit `localStorage`

## Folie 7: Vergleich

Angular:

- besser strukturiert
- Backend-Anbindung
- mehr Aufwand

Low-Code:

- schneller Prototyp
- einfacher zu starten
- weniger erweiterbar

## Folie 8: Lessons Learned

- KI/Low-Code spart Zeit bei Prototypen
- Angular ist besser für richtige Anwendungen
- Tests und Doku sind wichtig

## Folie 9: Fazit

Die Angular-App eignet sich besser als echte Anwendung.
Die Low-Code-App eignet sich besser, um schnell eine Idee zu zeigen.
